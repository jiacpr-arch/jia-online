# แผน: เกมจำลองสถานการณ์แบบ Code Blue Sim บน cpr.morroo.com

> เอกสารวางแผน (design doc) — ยังไม่ใช่โค้ด
> คำถามตั้งต้น: *"เอาเกมแบบ Code Blue Sim (จาก acls/bls.morroo.com) มาทำใน cpr.morroo.com
> โดยใช้ความรู้จากเนื้อหาบทเรียนของเว็บเป็นเนื้อหาเกม — ทำได้ไหม?"*
>
> **คำตอบ: ทำได้ และทำได้เร็วกว่าที่คิด** เพราะของที่ต้องใช้มีอยู่แล้วเกือบหมด

---

## 1. ทำไมถึงทำได้ง่าย — ของที่มีอยู่แล้ว

| ของที่ต้องใช้ | มีอยู่ที่ไหน | สภาพ |
|---|---|---|
| **Engine เกม** (state, คะแนน, combo, HP, เกรด S/A/B/C) | `acls-emr/src/game/storyEngine.js` | ~200 บรรทัด **logic ล้วน ไม่มี React/DOM/dependency ใดๆ** — copy มาใช้ได้ตรงๆ |
| **UI เกมสไตล์ visual novel** (กล่องบทพูด, ตัวเลือก, ตะโกนเต็มจอ, ECG, debrief) | `acls-emr/src/pages/CodeBlueSim.jsx` + `codeBlueSim.css` | ดัดแปลงได้ — ต้องตัดส่วนที่ผูกกับระบบ ACLS (react-router, zustand store, cohort sync) ออก |
| **เคสต้นแบบเนื้อหา BLS** 10 เคส | `acls-emr/src/data/scenarios/bls*.js` | เนื้อหาใกล้บทเรียน cpr.morroo.com มาก ใช้เป็นโครงดัดแปลงเป็น "ฉบับประชาชน" ได้ |
| **เนื้อหาความรู้** (จุดตัดสินใจในเกม) | `jia-online/src/App.jsx` — `COURSE.modules` บท 1–6 + quiz ทุกข้อ | ครบแล้ว: ลึก 5–6 ซม., 100–120/นาที, 30:2, ตำแหน่งแผ่น AED, "ฉันถอย คุณถอย ทุกคนถอย!" ฯลฯ |
| **โครงเว็บปลายทาง** | `jia-online` = React 18 + Vite เหมือน acls-emr | import engine ได้ทันที ไม่ต้องเพิ่ม dependency ใดๆ |

หลักฐานว่าแนวทาง "engine เดียว หลายเว็บ" ใช้ได้จริง: **bls.morroo.com ทำแบบนี้อยู่แล้ว**
(acls-emr มี `VITE_COURSE_MODE=bls` → เกมเดียวกัน คนละชุดโจทย์ คนละแบรนด์)

## 2. ทางเลือกสถาปัตยกรรม — เลือกทาง A

### ทาง A (แนะนำ): port เกมเข้าไปเป็นหน้าหนึ่งใน jia-online
- ผู้เรียนอยู่ในเว็บเดียวตลอด ไม่หลุดออกไป subdomain อื่น
- Pixel / PostHog / funnel ที่มีอยู่ วัดผลเกมได้ทันทีในที่เดียว
- ใช้ระบบ progress + signup gate + certificate เดิมของเว็บผูกกับเกมได้
- ต้นทุน: copy engine (~200 บรรทัด) + เขียน UI ใหม่แบบย่อ + เขียนเคสใหม่ 6 เคส

### ทาง B (ทางลัด แต่ไม่ตรงโจทย์): เพิ่มโหมด `cpr` ใน acls-emr แล้ว deploy แยก
- แบบเดียวกับ bls.morroo.com → ได้เกมเต็มรูป (เสียง, achievements, leaderboard) เร็วมาก
- แต่ผู้เรียนต้องเด้งไปอีกโดเมน (เช่น sim.cpr.morroo.com) — ประสบการณ์ขาดตอน, tracking แยกกัน
- เก็บไว้เป็น fallback ถ้าอยากได้ของเล่นได้ภายในวันเดียว

**ข้อสรุป: ทาง A** เพราะโจทย์คือ "ทำใน cpr.morroo.com โดยใช้เนื้อหาของเว็บ"

## 3. จุดต่างสำคัญที่ต้อง "แปลง" ไม่ใช่แค่ copy

ผู้เรียน cpr.morroo.com คือ**ประชาชนทั่วไป (lay rescuer)** แต่เคส BLS เดิมเขียนสำหรับ
บุคลากรทางการแพทย์ — ต้องแปลงเนื้อหาดังนี้:

| ของเดิม (BLS provider) | ฉบับประชาชน (cpr.morroo.com) |
|---|---|
| คลำชีพจร carotid พร้อมดูการหายใจ ≤10 วิ | **ดูการหายใจอย่างเดียว** (ไม่หายใจ/เฮือก = เริ่มปั๊ม) — ตรงกับ quiz บท 1 ของเว็บ |
| ทีมกู้ชีพในโรงพยาบาล (พยาบาลมิ้นท์, หมอฝน) | **พลเมืองดี**: ผู้เล่นเป็นคนแรกที่เจอเหตุ + พนักงานห้าง + ญาติผู้ป่วย + **เสียงเจ้าหน้าที่ 1669 ปลายสาย**คอยแนะนำ (ตัวช่วยแบบ hint ในตัว) |
| ศัพท์แพทย์ (ROSC, rhythm, defib) | ภาษาชาวบ้าน: "หายใจแล้ว!", "เครื่อง AED สั่งช็อก" — AED เป็นคนวิเคราะห์ ผู้เล่นไม่ต้องอ่านคลื่นหัวใจ |
| ยา/ใส่ท่อ/monitor | ตัดออกทั้งหมด — ขอบเขตแค่ ประเมิน → โทร 1669 → CPR → AED → ส่งต่อทีมกู้ชีพ |

ECG strip มุมจอของเดิม → เปลี่ยนเป็น **จอเครื่อง AED + เสียงพูดของเครื่อง** ในเคสที่มี AED
(ยังใช้ component `EcgStrip` เดิมได้ในฉากที่เครื่องวิเคราะห์)

## 4. แผนที่เกม ↔ บทเรียน — เคสละบท จุดตัดสินใจมาจาก quiz ตรงๆ

| เคสในเกม | อิงบทเรียน | ต้นแบบจาก acls-emr | ตัวอย่างจุดตัดสินใจ (จาก quiz ของเว็บ) |
|---|---|---|---|
| 1. "ล้มหมดสติในห้าง" | บท 1: CPR ผู้ใหญ่ | `blsCollapse` + `blsHandsOnly` | Scene safety ก่อน → ตบบ่าตะโกนเรียก → ชี้คนโทร 1669/เอา AED → กดลึก 5–6 ซม. เร็ว 100–120/นาที |
| 2. "ทารกนิ่งไปในเปล" | บท 2: CPR ทารก | `blsInfantCpr` | ใช้ 2 นิ้ว/2 นิ้วโป้ง, ลึก ~4 ซม., 30:2 (คนเดียว) → 15:2 (สองคน), ตำแหน่งใต้แนวราวนม |
| 3. "สำลักกลางวงหมูกระทะ" | บท 3: Choking ผู้ใหญ่ | `blsChoking` | ไอได้เสียงดัง = ห้ามตบหลัง → อุดกั้นสมบูรณ์ = ตบหลัง/กดท้อง (Heimlich เหนือสะดือ) → หมดสติ = วางลงพื้นเริ่ม CPR |
| 4. "ทารกสำลักนมผสมของเล่น" | บท 4: Choking ทารก | `blsInfantChoking` | คว่ำหน้าศีรษะต่ำ ตบหลัง 5 ครั้ง สลับพลิกหงายกดอก 5 ครั้ง — ห้ามกดท้อง ห้ามล้วงคอ ห้ามจับขาสะบัด |
| 5. "AED ข้างสระว่ายน้ำ" | บท 5: CPR & AED ผู้ใหญ่ | `blsAedWet` | ลากพ้นน้ำ เช็ดอกให้แห้ง → แผ่นใต้ไหปลาร้าขวา + ใต้ราวนมซ้าย → "ฉันถอย คุณถอย ทุกคนถอย!" → ปั๊มต่อทันทีหลังช็อก → recovery position |
| 6. "เด็กจมน้ำ / ทารกกับ AED" | บท 6: CPR & AED เด็ก/ทารก | `blsChildDrowning` | แผ่น AED เด็กติดหน้า-หลัง, ไม่มีแผ่นเด็กใช้แผ่นผู้ใหญ่ได้ (อย่าให้แผ่นแตะกัน), 15:2 เมื่อมี 2 คน |
| 7. โบนัส "Final Rescue" (ปลดหลังสอบผ่าน) | รวมทุกบท | เขียนใหม่ | เคสยาวรวมทุกทักษะ — ของรางวัลคู่ certificate + ปุ่มแชร์ |

ทุกจุดตัดสินใจตอบผิด → มีคำอธิบาย "ทำไมผิด" ที่**โยงกลับไปบทเรียน/วิดีโอของเว็บ** (debrief ท้ายเกมมีปุ่ม "ดูวิดีโอบทนี้ซ้ำ")

## 5. Game design ที่คงไว้จาก Code Blue Sim

- **Visual novel + จุดตัดสินใจมีผลจริง**: ตอบผิด → แถบ Patient Stability (HP) ลด, สถานการณ์แย่ลง, หมดแถบ = ช่วยไม่ทัน (จบแบบ mortality review ไม่ใช่แค่คะแนนน้อย)
- **เวลาตัดสินใจจำกัด + ตำแหน่งตัวเลือกสุ่ม** — ความกดดันเหมือนเหตุจริง
- **Interjection ตะโกนเต็มจอ**: "มีคนหมดสติ!!", "หัวใจหยุดเต้น!!", "เครื่องสั่งช็อก!!", "เขาหายใจแล้ว!!" + จอสั่น/สั่นเครื่อง
- **ระดับความยาก** easy/normal/hard (เวลาตัดสินใจ 30/20/12 วิ, HP 7/5/3, hint เปิด/ปิด) — ใช้ `DIFFICULTY` เดิมจาก engine
- **คะแนน**: ความแม่นยำ + โบนัสความไว + คอมโบ ×1.5, เกรด S/A/B/C, hiscore ต่อระดับความยาก (localStorage)
- **Debrief ท้ายเกม**: timeline สิ่งที่ทำ, จุดที่พลาดพร้อมคำอธิบาย, ลิงก์กลับบทเรียน
- เฟสหลัง: เสียง (beep, metronome, เสียงเครื่อง AED), haptics ตอนช็อก/ตอบผิด

## 6. การวางในเว็บ (UX flow)

```
Landing ──► เล่นเคสตัวอย่างฟรี (เคส 1 ฉบับสั้น) ─► ชวนสมัครเรียน   ← ตัวดัน conversion
Course  ──► ผ่าน quiz บทที่ N ─► ปุ่ม "ลองของจริง! ภารกิจบทที่ N" ─► เกมเคสที่ N
        ──► เมนู "เกมฝึกสถานการณ์" (hub รวม 6 เคส, ล็อกตามบทที่ยังไม่ผ่าน)
Final Exam ผ่าน ─► ปลดเคสโบนัส "Final Rescue" ─► จอฉลอง + แชร์ (ใช้ html-to-image ที่มีอยู่)
```

- ปลดล็อกเคสตาม progress ของบทเรียน → เกมเป็นรางวัล + แบบฝึกทวน ไม่ใช่ทางลัดข้ามบทเรียน
- ผลเกมเก็บ localStorage ก่อน (แบบเดิมของ Code Blue Sim) — เฟสหลังค่อย sync ขึ้น Supabase ที่มีอยู่แล้วถ้าอยากทำ leaderboard

## 7. โครงสร้างโค้ดใน jia-online

```
src/
  game/
    storyEngine.js     ← copy จาก acls-emr (แทบไม่ต้องแก้ — logic ล้วน)
    characters.js      ← ตัวละครชุดใหม่: ผู้เล่น, พลเมืองดี, พนักงานห้าง, ญาติ, เสียง 1669
    GamePage.jsx       ← UI เกม (ดัดแปลงจาก CodeBlueSim.jsx: ตัด router/store/cohort, มือถือเป็นหลัก)
    game.css           ← ดัดแปลงจาก codeBlueSim.css
    scenarios/
      lesson1Adult.js … lesson6ChildAed.js + finalRescue.js   (7 เคส "ฉบับประชาชน")
  App.jsx              ← เพิ่ม case "game"/"gamehub" ใน switch + ปุ่มเข้าเกม (~30 บรรทัด)
```

หมายเหตุ: jia-online เขียนแบบไฟล์เดียว แต่ Vite import ไฟล์แยกได้ปกติ (มี `src/main.jsx` อยู่แล้ว) — โค้ดเกมแยกโฟลเดอร์จะไม่ทำให้ `App.jsx` บวมขึ้นอีก

## 8. แผนงานเป็นเฟส

### Phase 1 — โครงเกมเล่นได้จริง 1 เคส (~1–2 วัน)
- [ ] copy `storyEngine.js` + เขียน `GamePage.jsx` + `game.css` ฉบับย่อ (มือถือเป็นหลัก)
- [ ] ตัวละครชุดประชาชน (ใช้ SVG แบบเดียวกับ `CharacterSprite` เดิมไปก่อน)
- [ ] เคสบทที่ 1 "ล้มหมดสติในห้าง" ฉบับ lay rescuer เล่นจบได้ มี debrief
- [ ] ปุ่มเข้าเกมจากหน้า Course หลังผ่าน quiz บท 1

### Phase 2 — ครบ 6 เคส + ผูกกับ progress (~2–3 วัน, งานหลักคือเนื้อหา)
- [ ] เขียนเคส 2–6 ตามตารางข้อ 4 — **ครูผู้สอนรีวิวความถูกต้องทุกเคสก่อนขึ้นจริง**
- [ ] hub เกม + ปลดล็อกตามบทเรียน + hiscore/เกรดต่อเคส
- [ ] debrief โยงกลับวิดีโอบทเรียน

### Phase 3 — Polish + ดัน conversion (~1–2 วัน)
- [ ] เสียง + haptics + เสียงเครื่อง AED
- [ ] เคสตัวอย่างฟรีหน้า Landing + event Pixel/PostHog (game_start, game_clear, game_share)
- [ ] เคสโบนัส "Final Rescue" หลังสอบผ่าน + การ์ดแชร์ผล (html-to-image)

### Phase 4 — ทางเลือกต่อยอด
- [ ] Leaderboard บน Supabase (ตารางใหม่ + RLS แบบเดียวกับของเดิม)
- [ ] เคสเพิ่ม: คนท้องสำลัก, จมน้ำผู้ใหญ่, opioid ฯลฯ (ต้นแบบมีใน acls-emr แล้ว)
- [ ] ระบบเขียนโจทย์เอง / AI แต่งโจทย์ ตามแผน `acls-emr/docs/code-blue-sim-v2-plan.md`

## 9. คำถามที่ควรตัดสินใจก่อนเริ่ม Phase 1

1. **การเข้าถึงเกม**: ปลดล็อกตามบทเรียน (แนะนำ — ดันคนเรียนให้จบ) หรือเล่นฟรีทุกเคสตั้งแต่แรก?
2. **ต้องสมัคร/ล็อกอินก่อนเล่นไหม**: ผูกกับ signup gate ที่มีอยู่ (เกมเป็นเหยื่อล่อสมัคร) หรือเล่นได้เลย?
3. **ตัวละคร**: ชุดใหม่ "พลเมืองดี" (แนะนำ — ตรงกลุ่มเป้าหมาย) หรือยกทีมพยาบาลมิ้นท์จาก acls มาเลยเพื่อความคุ้นตาแบรนด์ morroo?
4. **ชื่อเกม**: เช่น "CPR HERO — ภารกิจพลเมืองดี" / "หน่วยกู้ชีพมือใหม่" — ใช้โปรโมตในเพจ/โฆษณาได้
